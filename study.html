02_엘리먼트?

요소 노드, 어트리뷰트 노드, 텍스트 노드로 만들어지는 가상 dom의 개념

03_컴포넌트?

엘리먼트를 반환하는 개념을 가진 화면의 구성 단위

함수형 컴포넌트에서만 훅을 사용할 수 있다

<script type="text/javascript">
	function HelloWorld(){
       return React.createElement("h1", 
       {classname: "greeting", id:"test"}, 
       "hello word function");
   };

   ReactDOM.createRoot(document.getElementById("root")).render(React.createElement(HelloWorld));
</script>

createElement를 이용해 엘리먼트를 정의하면 복잡해지며 가독성 또한 좋지 않다.

그렇기에 ReactTeam은 ReactElement를 정의하는 JSX 문법을 제공한다.

04_JSX란?

JavaScript의 확장된 문법으로, ReactElement를 xml 문법 형식으로 정의할 수 있는 방법이다.

단, JSX는 공식적인 자바스크립트 문법은 아니며 바벨이라는 트랜스 파일링 툴이 필요하다.

05_babel이란?

여러가지 랭귀지들을 하나로 컴파일할 수 있게 해주는 역할

바벨은 ES6+, E5 next 문법을 ES5문법으로 변환해 주는 도구이다.

  • 이 안에 들어가는 모든 엘리먼트를 ReactDOM에서 관리하기 떄문에 이것을 루트(root)DOM 혹은 루트 node라고 부른다.
  • 일반적으로 react로 구현된 애플리케이션은 하나의 루트 DOM이 있다.
  • 단, 리액트를 기존 앱에 통합하려는 경우 원하는 만큼 독립된 루트 DOM노드를 가질 수도 있다.

바벨 적용 방법(CDN)

  1. 바벨 CDN 구문을 추가한다.
    <!--babel-->
<script crossorigin src="https://unpkg.com/@babel/standalone/babel.min.js"></script>
  1. script 태그에 type 속성에 text/babel 속성을 추가한다.
    <script type="text/babel">
</script>

06_JSX 사용 규칙

동일 레벨의 노드들이 연달아 정의되는 경우 문법적인 에러가 발생하게 된다.

여러 형제 레벨의 엘리먼트를 정의해야하는 경우 (div 태그와 같은) 하나의 부모 엘리먼트로 감싸야 하는데 div 요소 같은 실질적인 요소를 사용하지 않으려면 Fragment를 이용할 수 있다. (react 16이전 버전에 추가됨)

  1. 최상위 엘리먼트가 두 개 이상이면 에러가 발생한다
    <script type="text/babel">
		const element = (
		    <h1>안녕</h1>
        <h1>하세요</h1>
	  )
		ReactDOM.createRoot(document.getElementById("root")).render(elemet);
</script>
  1. <div> 태그로 감싸서 하나의 돔트리를 생성할 수 있도록 한다.
    <script type="text/babel">
		const element = (
		    <div>
		        <h1>안녕</h1>
            <h1>phone : {user.phone}</h1>
        </div>
    )
		ReactDOM.createRoot(document.getElementById("root")).render(elemet);
</script>
  1. <React.Fragment>로 감싸서 형식상의 돔 트리상 상위 엘리먼트를 만들어준다.
    • 의미없는 태그가 길어진다는 단점
    <script type="text/babel">
		const elemet = (
		    <React.Fragment>
		        <h1>안녕{user.name}</h1>
		        <h1>phone : {user.phone}</h1>
		    </React.Fragment>
    );
		ReactDOM.createRoot(document.getElementById("root")).render(elemet);
</script>
  1. ReactLibrary로부터 Fragment 객체만 구조분해 할당을 해주면 React. 을 생략할 수 있다.
    • 플래그먼트의 축약 표현
    <script type="text/babel">
		const {Fragment} = React;
    const {name, phone} = user;
    const elemet = (
		    <Fragment>
		        <h1>안녕 {name + "님"}</h1>
            <h1>phone : {phone + "로 전화해도 됨?"}</h1>
		    </Fragment>
    );
		ReactDOM.createRoot(document.getElementById("root")).render(elemet);
</script>

10_클래스형 컴포넌트 (사용자 정의 태그)

<!--
        01. 클래스형 컴포넌트 (사용자 정의 태그)
        
        ES6에서 제공되는 클래스 문법을 이용해 렌더링 될 컴포넌트를 정의하는 방법을 제공한다.
        render() 함수는 반드시 작성되어야 하며,
        반환값으로는 렌더링 될 리액트 엘리먼트를 정의한다. (일반적으로 JSX를 사용함)

        JSX만으로도 리액트 엘리먼트를 정의할 수 있지만 클래스형으로 컴포넌트를 작성하게 되면 여러가지 장점이 있다.
        중복되는 엘리먼트들을 추상화해서 컴포넌트를 작성하고,
        ★★★작성된 컴포넌트를 재사용하는 것이 리액트가 추구하는 한가지 방향이다.★★★

        컴포넌트를 정의하는 방법은 현재 두 가지 방식을 제공한다.
        클래스형 컴포넌트와 함수형 컴포넌트이다.
        사용하는 것은 함수형 컴포넌트가더 간결하고 선언하기도 편하며, 메모리도 덜 사용하기에 더 권자오디는 방식이다.
        그러나 state 사용 및 라이프 사이클기능, 임의의 메소드 정의 등은 클래스 컴포넌트에서만 사용 가능한 방식이다.
        이러한 함수형 컴포넌트의 기능적 한계로 훅스(hooks)를 제공하고 있다.

        리액트 팀에서는 함수형 컴포넌트 사용을 권장하고 있으나
        클래스형 컴포넌트를 없애지 않고 유지하는 방향으로 선택하고 있다.
-->
<script type="text/babel">
        /*
        클래스형 컴포넌트 사용법
        1. 앞글자는 대문자로
        2. 닫기 태그 필수
        3. 클래스명과 태그명을 동일하게

        사용자 정의 태그라고 생각을 하면 된다.*/
        class Title extends React.Component { //1. Title : 대문자로 시작
            render() {
                return(
                    <h1>
                        Class Component
                    </h1> //2. 닫기 태그 필수
                );
            }
        }
        ReactDOM.createRoot(document.getElementById("root")).render(<Title/>); //2. 닫기 태그 필수, 3. Title 클래스명과 태그명을 동일하게
</script>

11_함수형 컴포넌트 (사용자 정의 태그)

<!--
        02. 함수형 컴포넌트 (사용자 정의 태그)

        함수형 컴포넌트는 간결하게 작성 가능하다.
        함수의 반환값으로 ReactElement만 정의해서 반환하면 된다. (일반적으로 JSX 사용)

        함수형 컴포넌트 작성 시 유의 사항은 앞 글자를 대문자로 정의해야 한다는 것이다.
        사용자 정의 엘리먼트를 이용할 시 <Title/> 이런 형태로 사용하게 되는데,
        앞글자가 소문자이면 html로 인식하게 되고 존재하지 않는 html 태그 엘리먼트이기 때문에 제대로 그려지지 않는다.

        함수형 컴포넌트의 단점은 state와 라이프사이클 api 사용이 불가능하다는 것이었으나,
        v16.8부터 hooks를 제공하면서 해당 문제가 해결되었다.
-->
<script type="text/babel">
        function Title() {
            return (
                <h1>
                    Function component
                </h1>
            );
        }
        ReactDOM.createRoot(document.getElementById("root")).render(<Title/>);
</script>
